Create Your Own Plugin

Qodalis CLI has a rich plugin architecture that lets you add custom commands. The easiest way to get started is with the scaffolding tool, which generates all the boilerplate for you.

Scaffold a New Plugin

Run the scaffolding tool interactively:

shell 
npx @qodalis/create-cli-plugin

Or use non-interactive mode with all options:

shell 
npx @qodalis/create-cli-plugin --name weather --description "Weather forecasts" --processor-name WeatherProcessor

Plugin Structure

The scaffolding tool generates the following structure:

Generated files 
cli-weather/
  src/
    lib/
      processors/
        cli-weather-command-processor.ts
    public-api.ts
    cli-entrypoint.ts
  tsup.config.ts
  package.json
File Purpose
cli-weather-command-processor.ts The command processor with your logic
public-api.ts Public exports for the package
cli-entrypoint.ts IIFE entry point for runtime installation via pkg add
tsup.config.ts Build configuration producing CJS, ESM, and IIFE bundles

Implementing the Processor

Every plugin command implements the ICliCommandProcessor interface. Here are the key fields and methods:

Field Description
command The command name users type (e.g. 'weather')
description Shown in help output and command listings
parameters Array of parameter descriptors with name, description, and required flag
processors Nested sub-command processors
processCommand(command, context) The handler that runs when the command is executed
cli-weather-command-processor.ts 
import {
    ICliCommandProcessor,
    ICliExecutionContext,
    CliProcessCommand,
} from '@qodalis/cli-core';

export class WeatherProcessor implements ICliCommandProcessor {
    command = 'weather';
    description = 'Get weather forecasts';

    parameters = [
        {
            name: 'city',
            description: 'City name to look up',
            required: true,
        },
        {
            name: '--units',
            description: 'Temperature units (metric or imperial)',
            required: false,
        },
    ];

    async processCommand(
        command: CliProcessCommand,
        context: ICliExecutionContext,
    ): Promise<void> {
        const city = command.args[0];
        if (!city) {
            context.writer.writeError('Please provide a city name.');
            return;
        }

        context.spinner.show(`Fetching weather for ${city}...`);
        // Your logic here
        context.spinner.hide();
        context.writer.writeln(`Weather in ${city}: Sunny, 22C`);
    }
}

Using the Execution Context

The ICliExecutionContext passed to processCommand gives you access to all the tools you need:

Service Usage
context.writer Write text, HTML, errors, and warnings to the terminal
context.spinner Show/hide a loading spinner
context.progressBar Display a progress bar for long operations
context.inputReader Read interactive input (text, password, confirm, select)
context.stateStore Persistent key-value storage across sessions
context.clipboard Read from and write to the clipboard

See the Execution Context page for full details.

Building

The generated tsup.config.ts builds your plugin in three formats: CommonJS, ESM, and IIFE (for browser use).

shell 
npx tsup

This produces:

  • dist/index.js — CommonJS bundle
  • dist/index.mjs — ESM bundle
  • dist/umd/index.js — IIFE bundle for standalone browser use
  • dist/index.d.ts — TypeScript declarations

Publishing

Publish your plugin to npm so others can install it:

shell 
npm publish --access public

Runtime Installation

One of the most powerful features of Qodalis CLI is runtime plugin installation. Users can install your published plugin directly from the terminal without rebuilding their application:

Terminal 
pkg add @qodalis/cli-weather

This downloads the IIFE bundle from npm and loads it dynamically into the running CLI. No application rebuild or deployment is required.
