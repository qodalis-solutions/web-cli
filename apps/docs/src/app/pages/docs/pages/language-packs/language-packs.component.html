Language Packs

Qodalis CLI supports internationalization (i18n) through language pack plugins. Each language pack translates the built-in CLI interface strings (help text, error messages, prompts) into the target language.

Available Language Packs

Language Package Install
{{ pack.name }} {{ pack.npmPackage }} npm install {{ pack.npmPackage }}

Usage

Install a language pack and add it to your modules array:

npm 
npm install @qodalis/cli-lang-es
Import and use 
import { langEsModule } from '@qodalis/cli-lang-es';

// Angular: <cli [modules]="[langEsModule]" />
// React:   <Cli modules={[langEsModule]} />
// Vue:     <Cli :modules="[langEsModule]" />

You can also install language packs at runtime without rebuilding:

Terminal 
pkg add @qodalis/cli-lang-es

Creating a Language Pack

Language packs are regular CLI plugins that register translations during initialization. The plugin's onInit hook receives the execution context, which provides access to the translation service.

How It Works

  • Language packs register translations via ICliTranslationService
  • Use translator.addTranslations('es', translations) in the onInit hook
  • Translation keys follow the cli.<command>.<key> pattern
lang-pack-example.ts 
export class SpanishLanguageProcessor implements ICliCommandProcessor {
    command = '__lang_es__';
    description = 'Spanish language pack';

    onInit(context: ICliExecutionContext): void {
        const translator = context.translationService;
        translator.addTranslations('es', {
            'cli.help.description': 'Muestra la ayuda',
            'cli.clear.description': 'Limpia la terminal',
            'cli.version.description': 'Muestra la version',
            // ... more translations
        });
    }
}
