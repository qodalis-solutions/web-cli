Node.js Server

The Node.js backend is an Express-based server with TypeScript support. It uses node-pty for interactive shell sessions and ws for WebSocket connections.

Installation

npm 
npm install @qodalis/cli-server

Basic Setup

server.ts 
import { createCliServer } from '@qodalis/cli-server';

const { app, builder } = createCliServer({ port: 8047 });
builder.addProcessor(new MyProcessor());
app.listen(8047);

Adding Processors

Register command processors using the fluent builder API:

processors.ts 
builder.addProcessor(new EchoProcessor());
builder.addProcessor(new HashProcessor());

// Or register an entire module
builder.addModule(new WeatherModule());

Docker

shell 
docker compose up cli-server-node

The server runs on port 8047 by default.

Connecting the Frontend

app.component.ts 
import { CliOptions } from '@qodalis/cli-core';

const options: CliOptions = {
    serverUrl: 'http://localhost:8047',
};

// Angular: <cli [options]="options"></cli>
// React:   <Cli options={options} />
// Vue:     <Cli :options="options" />

Python Server

The Python backend is built with FastAPI and uvicorn. It uses the native pty module for interactive shell sessions and FastAPI's built-in WebSocket support.

Installation

pip 
pip install qodalis-cli-server

Basic Setup

server.py 
from qodalis_cli import create_cli_server

result = create_cli_server()
builder = result.builder
builder.add_processor(MyProcessor())

Adding Processors

Register command processors using the builder API:

processors.py 
builder.add_processor(EchoProcessor())
builder.add_processor(HashProcessor())

# Or register an entire module
builder.add_module(WeatherModule())

Docker

shell 
docker compose up cli-server-python

The server runs on port 8048 by default.

Connecting the Frontend

app.component.ts 
import { CliOptions } from '@qodalis/cli-core';

const options: CliOptions = {
    serverUrl: 'http://localhost:8048',
};

// Angular: <cli [options]="options"></cli>
// React:   <Cli options={options} />

.NET Server

The .NET backend is built with ASP.NET Core. It includes Swagger UI for API exploration and distributes as a NuGet package.

Installation

NuGet 
dotnet add package Qodalis.Cli

Basic Setup

Program.cs 
var builder = WebApplication.CreateBuilder(args);

// Add CLI services
builder.Services.AddCli();

var app = builder.Build();

// Enable CLI middleware
app.UseCli();

app.Run();

Adding Processors

Register command processors via dependency injection:

Program.cs 
builder.Services.AddCli(options => {
    options.AddProcessor<EchoProcessor>();
    options.AddProcessor<HashProcessor>();
    options.AddModule<WeatherModule>();
});

Docker

shell 
docker compose up cli-server-dotnet

The server runs on port 8046 by default.

Connecting the Frontend

app.component.ts 
import { CliOptions } from '@qodalis/cli-core';

const options: CliOptions = {
    serverUrl: 'http://localhost:8046',
};

// Angular: <cli [options]="options"></cli>
// React:   <Cli options={options} />
// Vue:     <Cli :options="options" />
