Configuration

Qodalis CLI can be configured at build time through component properties or at runtime using built-in config commands.

CliOptions

The following options can be passed to the CLI component:

Option Type Default Description
logLevel CliLogLevel INFO Log verbosity level (DEBUG, INFO, WARN, ERROR)
welcomeMessage string (default) Custom welcome message displayed when the terminal opens
theme string 'default' Terminal color theme
serverUrl string null Server connection URL for remote command execution

Framework Configuration

Angular Configuration

Pass options via the [options] input binding:

app.component.ts 
import { CliOptions } from '@qodalis/angular-cli';

cliOptions: CliOptions = {
    logLevel: 'DEBUG',
    welcomeMessage: 'Welcome to my app!',
    theme: 'default',
    serverUrl: 'http://localhost:8046',
};
app.component.html 
<cli [options]="cliOptions" [modules]="[]"></cli>

<!-- Or with the panel variant -->
<cli-panel [options]="cliOptions" [modules]="[]"></cli-panel>

Don't forget to add the styles to your angular.json:

angular.json 
"styles": [
    "node_modules/@qodalis/angular-cli/src/assets/styles.sass"
]

React Configuration

Pass options through CliConfigProvider props:

App.tsx 
import { Cli, CliPanel, CliConfigProvider } from '@qodalis/react-cli';

function App() {
    const options = {
        logLevel: 'DEBUG',
        welcomeMessage: 'Welcome to my app!',
        theme: 'default',
        serverUrl: 'http://localhost:8046',
    };

    return (
        <CliConfigProvider modules={[]} options={options}>
            <Cli />
        </CliConfigProvider>
    );
}

Vue Configuration

Pass options through CliConfigProvider props:

App.vue 
<script setup>
import { Cli, CliPanel, CliConfigProvider } from '@qodalis/vue-cli';

const options = {
    logLevel: 'DEBUG',
    welcomeMessage: 'Welcome to my app!',
    theme: 'default',
    serverUrl: 'http://localhost:8046',
};
</script>

<template>
    <CliConfigProvider :modules="[]" :options="options">
        <Cli />
    </CliConfigProvider>
</template>

Runtime Configuration

You can view and modify configuration at runtime using the built-in config command directly in the terminal:

List all configuration values 
config list
Get a specific value 
config get theme
Set a value 
config set theme dark
Delete a value 
config delete theme
