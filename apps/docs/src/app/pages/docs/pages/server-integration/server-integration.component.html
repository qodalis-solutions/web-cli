Server Integration

Qodalis CLI can connect to a backend server to execute commands on the server side, access the filesystem, stream events, and open interactive shell sessions. The frontend CLI communicates with the backend via a standard REST and WebSocket API.

Architecture

The integration follows a simple client-server pattern. The frontend CLI component sends commands to the backend server, which executes them and returns the output. Real-time features like event streaming and PTY shell sessions use WebSocket connections.

Architecture 
Frontend CLI  <-- REST / WebSocket -->  Backend Server
(Browser)                                 (Node.js / Python / .NET)
  |                                          |
  |-- POST /api/v1/cli/execute ----------->  |-- Execute command
  |-- GET  /api/v1/cli/commands -----------> |-- List commands
  |-- WS   /ws/v1/cli/shell ---------------> |-- PTY shell session
  |-- WS   /ws/v1/cli/events --------------> |-- Event streaming
  |-- GET  /api/cli/fs/* ------------------> |-- Filesystem access

Common API

All three backend servers implement the same API surface:

Endpoint Method Purpose
/api/cli/versions GET Version discovery — supported versions, preferred version
/api/v1/cli/commands GET List all registered command processors
/api/v1/cli/execute POST Execute a CLI command on the server
/ws/v1/cli/events WS Server-push event streaming
/ws/v1/cli/shell WS Interactive PTY shell session
/api/cli/fs/* Various Filesystem operations (ls, cat, stat, upload, download, mkdir, rm)
