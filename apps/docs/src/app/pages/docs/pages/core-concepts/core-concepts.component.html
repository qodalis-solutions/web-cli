Command Processors

Every command in Qodalis CLI is powered by a command processor. A processor implements the ICliCommandProcessor interface and defines the command name, description, parameters, and the logic that runs when the command is executed.

The ICliCommandProcessor Interface

The key fields of a command processor are:

Field Type Description
command string The command name that triggers this processor (e.g. 'greet')
description string Shown in help output and command listings
parameters ICliCommandParameterDescriptor[] Describes accepted flags and arguments
processors ICliCommandProcessor[] Nested sub-command processors (e.g. git commit, git push)
extendsProcessor string Command name to wrap/extend with additional sub-commands

The processCommand Method

The processCommand(command, context) method is called when the user types the command. It receives the parsed command and an execution context that provides access to terminal I/O, state, spinner, and more.

Example Implementation

greet-processor.ts 
export class GreetProcessor implements ICliCommandProcessor {
    command = 'greet';
    description = 'Say hello';

    parameters = [
        {
            name: 'name',
            description: 'Name to greet',
            required: false,
        },
    ];

    async processCommand(
        command: CliProcessCommand,
        context: ICliExecutionContext,
    ): Promise<void> {
        const name = command.args[0] || 'World';
        context.writer.writeln(`Hello, ${name}!`);
    }
}

Sub-Commands

Processors can define nested processors to create command hierarchies. For example, a git processor can have commit and push as sub-command processors.

nested-processor.ts 
export class GitProcessor implements ICliCommandProcessor {
    command = 'git';
    description = 'Git commands';

    processors = [
        new GitCommitProcessor(),
        new GitPushProcessor(),
    ];
}

Extending Existing Commands

Use extendsProcessor to add sub-commands to an existing processor without modifying it. This is useful for plugins that want to extend built-in commands.

extend-processor.ts 
export class GitStashProcessor implements ICliCommandProcessor {
    command = 'stash';
    description = 'Stash changes';
    extendsProcessor = 'git'; // Adds 'stash' as a sub-command of 'git'

    async processCommand(
        command: CliProcessCommand,
        context: ICliExecutionContext,
    ): Promise<void> {
        context.writer.writeln('Changes stashed.');
    }
}

Execution Context

The ICliExecutionContext is passed to every command processor when it executes. It provides access to all the services and utilities you need during command execution.

Context Properties

Property Type Description
writer ICliTerminalWriter Write text, HTML, and formatted output to the terminal
spinner ICliSpinner Show a loading spinner with a message
progressBar ICliProgressBar Display a progress bar for long-running operations
inputReader ICliInputReader Read interactive input from the user
stateStore ICliStateStore Persistent key-value storage across sessions
clipboard ICliClipboard Read from and write to the system clipboard
abortSignal AbortSignal Signal to detect when the user cancels a command (Ctrl+C)
options CliOptions The CLI configuration options

Writing Output

The writer is the primary way to send output to the terminal.

writer-examples.ts 
// Write a line of text
context.writer.writeln('Hello, World!');

// Write styled HTML output
context.writer.writeHtml('<span style="color: green">Success!</span>');

// Write a warning or error
context.writer.writeError('Something went wrong');
context.writer.writeWarning('This is deprecated');

Spinner and Progress Bar

spinner-example.ts 
// Show a spinner while loading
context.spinner.show('Loading data...');
const data = await fetchData();
context.spinner.hide();

// Show a progress bar
context.progressBar.show({ total: 100 });
for (let i = 0; i <= 100; i++) {
    context.progressBar.update(i);
}
context.progressBar.hide();

Full-Screen Mode

Commands that need full control of the terminal (games, editors, pagers) can use full-screen mode:

fullscreen-example.ts 
// Enter full-screen mode
context.enterFullScreenMode();

// Render your full-screen UI...

// Exit when done
context.exitFullScreenMode();

Abort Signal

Long-running commands should check context.abortSignal to respond to user cancellation (Ctrl+C):

abort-example.ts 
while (!context.abortSignal.aborted) {
    // Do work...
    await sleep(100);
}
context.writer.writeln('Operation cancelled.');

Tabs & Split Panes

The cli-panel component supports multiple terminal sessions through tabs and horizontal split panes. Users can create, rename, duplicate, split, and close terminals — all within a single collapsible panel.

Tab Management

Each tab contains one or more terminal panes. Users interact with tabs through the tab bar at the top of the panel.

Action How
New tab Click the + button in the tab bar
Rename tab Double-click the tab title, type a new name, press Enter
Close tab Click the × button on the tab, or right-click → Close
Duplicate tab Right-click → Duplicate (preserves terminal state, history, and output)
Close others Right-click → Close Others / Close to the Right / Close All

Split Panes

Within a tab, you can split the terminal horizontally to run multiple sessions side by side.

Action How
Split right Right-click a tab → Split Right
Resize panes Drag the divider between panes
Focus a pane Click inside a pane (active pane shows an accent border)
Close a pane Click the × in the pane's top-right corner

Pane widths are distributed equally when a split is created. The minimum pane width is 10% of the panel. Closing the last pane in a tab closes the tab itself.

Panel Positions

The panel can be positioned at four edges of the viewport. The position cycles through:

bottom right top left

The position is persisted to localStorage and restored automatically.

Configuration

The cli-panel component accepts a CliPanelOptions object to configure panel behavior:

Option Type Default Description
isCollapsed boolean true Start collapsed or expanded
position string 'bottom' Panel edge: bottom, top, left, right
closable boolean true Show close button in panel header
resizable boolean true Allow drag-resizing the panel
syncTheme boolean false Sync panel chrome colors with the active terminal theme

Usage

Angular 
<cli-panel
    [options]="{{ '{' }} position: 'bottom', syncTheme: true {{ '}' }}"
    [modules]="modules"
></cli-panel>
React 
<CliPanel
    options={{ '{' }}{{ '{' }} position: 'bottom', syncTheme: true {{ '}' }}{{ '}' }}
    modules={{ '{' }}modules{{ '}' }}
/>
Vue 
<CliPanel
    :options="{{ '{' }} position: 'bottom', syncTheme: true {{ '}' }}"
    :modules="modules"
/>

State Preservation

When a tab is duplicated, the full terminal state is captured and restored in the new tab:

  • Terminal output buffer (serialized xterm content)
  • Command history
  • Processor state stores (todo items, config values, etc.)

Input Reader

The ICliInputReader interface provides methods for reading interactive input from the user. It is available through context.inputReader in every command processor.

Available Methods

Method Description
readLine(prompt) Read a line of text input
readPassword(prompt) Read input with masked characters (for passwords)
readConfirm(prompt) Ask a yes/no question, returns a boolean
readSelect(prompt, options) Show a list of options, user picks one
readSelectInline(prompt, options) Inline single-line select (compact UI)
readMultiSelect(prompt, options) Select multiple items from a list
readNumber(prompt) Read and validate a numeric value

Examples

Reading Text Input

readLine 
const name = await context.inputReader.readLine('What is your name?');
context.writer.writeln(`Hello, ${name}!`);

Reading a Password

readPassword 
const password = await context.inputReader.readPassword('Enter password:');
// Input is masked with dots

Confirmation Prompt

readConfirm 
const confirmed = await context.inputReader.readConfirm('Delete all files?');
if (confirmed) {
    context.writer.writeln('Deleting...');
}

Selection Menu

readSelect 
const color = await context.inputReader.readSelect('Pick a color:', [
    { label: 'Red', value: 'red' },
    { label: 'Green', value: 'green' },
    { label: 'Blue', value: 'blue' },
]);
context.writer.writeln(`You picked: ${color}`);

Multi-Select

readMultiSelect 
const toppings = await context.inputReader.readMultiSelect('Choose toppings:', [
    { label: 'Cheese', value: 'cheese' },
    { label: 'Pepperoni', value: 'pepperoni' },
    { label: 'Mushrooms', value: 'mushrooms' },
]);
context.writer.writeln(`Selected: ${toppings.join(', ')}`);

Reading a Number

readNumber 
const age = await context.inputReader.readNumber('Enter your age:');
context.writer.writeln(`You are ${age} years old.`);
